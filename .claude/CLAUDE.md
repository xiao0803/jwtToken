# 项目规则

## 项目背景
- 这是一个 Java Maven 项目
- 使用中文沟通，但代码、注释、提交信息遵循项目现有风格
- 优先保持最小改动

## 修改原则
1. 保持公开接口不变，除非我明确允许
2. 不修改无关文件
3. 优先复用已有工具类、测试风格、异常处理模式
4. 修改前先分析，修改后说明变更点
5. 如涉及数据库、配置、线程池、缓存，先提示风险

## 构建与测试
- 构建命令：mvn -q -DskipTests package
- 测试命令：mvn test
- 如测试失败，优先读取 .claude/runtime/failure-summary.txt

## 输出要求
- 先给"修改计划"
- 再执行修改
- 最后给"修改文件列表 + 风险 + 验证方法"
